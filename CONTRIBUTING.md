# Contributing to ASP.NET SPA Template

Thank you for your interest in contributing to the ASP.NET SPA Template! This document provides guidelines and instructions for contributing to the project.

## How to Contribute

### 1. Fork and Clone
1. Fork the repository on GitHub.
2. Clone your fork locally:
   ```bash
   git clone https://github.com/your-username/aspnet-spa-template.git
   cd aspnet-spa-template
   ```

### 2. Create a Branch
Create a descriptive branch for your feature or bug fix:
```bash
git checkout -b feature/your-feature-name
```

### 3. Development Requirements
- **.NET 10.0 SDK** is required to build and run the backend.

### 4. Run Tests
Ensure all tests pass before submitting a Pull Request:
```bash
dotnet test
```

### 5. Submit a Pull Request
1. Push your branch to your fork.
2. Open a Pull Request against the main branch of the original repository.
3. Provide a clear description of the changes and the problem they solve.

## Code Style
- Follow existing conventions in the codebase.
- Provide XML documentation comments for public APIs.
- **KEEP ALL author headers - DO NOT remove them**. If you significantly modify a file, you may add your name, but do not remove the original author headers.

## Issues
Use GitHub Issues to report bugs and suggest features. When reporting a bug, please include clear reproduction steps, expected behavior, actual behavior, and environment details to help us investigate.
