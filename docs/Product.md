# Product

The `Product` class represents a sellable item within the e-commerce domain of the `aspnet-spa-template` application. It encapsulates core inventory data, pricing information, categorization, and metadata required for display and transaction processing. Beyond simple data storage, the type includes computed properties for immediate availability checks and methods to safely modify stock levels, serving as the primary aggregate root for product-related operations.

### Methods

#### `ReduceStock`
```csharp
public void ReduceStock(int quantity)
```
Decrements the `StockQuantity` by the specified amount.
*   **Parameters**: `quantity` - The number of units to remove. Must be greater than zero.
*   **Returns**: void.
*   **Throws**: Throws an exception if `quantity` is less than or equal to zero, or if the requested amount exceeds the current `StockQuantity`.

#### `IncreaseStock`
```csharp
public void IncreaseStock(int quantity)
```
Increments the `StockQuantity` by the specified amount, typically used for returns or restocking.
*   **Parameters**: `quantity` - The number of units to add. Must be greater than zero.
*   **Returns**: void.
*   **Throws**: Throws an exception if `quantity` is less than or equal to zero.

## Usage

### Example 1: Creating and Initializing a Product
The following example demonstrates instantiating a new product, setting required fields, and verifying its initial purchase status.

```csharp
var newProduct = new Product
{
    Name = "Wireless Noise-Canceling Headphones",
    Description = "Premium audio experience with 20h battery life.",
    Price = 299.99m,
    StockQuantity = 50,
    Category = ProductCategory.Electronics,
    Sku = "WH-1000XM5-BLK",
    IsAvailable = true,
    CreatedAt = DateTime.UtcNow
};

// Ensure logic flags are consistent
newProduct.IsInStock = newProduct.StockQuantity > 0;
newProduct.CanPurchase = newProduct.IsAvailable && newProduct.IsInStock;

if (newProduct.CanPurchase)
{
    Console.WriteLine($"Product {newProduct.Name} is ready for sale.");
}
```

### Example 2: Managing Inventory Transactions
This example shows how to safely modify stock levels using the provided methods, handling potential errors when stock is insufficient.

```csharp
public void ProcessOrder(Product product, int requestedQuantity)
{
    try
    {
        // Attempt to reserve stock
        product.ReduceStock(requestedQuantity);
        
        // Update derived flags immediately after mutation
        product.IsInStock = product.StockQuantity > 0;
        product.CanPurchase = product.IsAvailable && product.IsInStock;
        
        Console.WriteLine($"Order processed. Remaining stock: {product.StockQuantity}");
    }
    catch (ArgumentException ex)
    {
        // Handle cases where quantity is invalid or insufficient stock exists
        Console.WriteLine($"Failed to process order: {ex.Message}");
        throw;
    }
}

public void ProcessReturn(Product product, int returnQuantity)
{
    // Restock items
    product.IncreaseStock(returnQuantity);
    product.IsInStock = true; // Logic assumes return makes it in stock
    product.CanPurchase = product.IsAvailable;
}
```

## Notes

*   **Data Consistency**: The boolean properties `IsInStock` and `CanPurchase` appear to be settable properties rather than read-only computed getters. Callers must ensure these flags are manually updated immediately after modifying `StockQuantity`, `IsAvailable`, or `Price` to prevent state desynchronization.
*   **Thread Safety**: The `ReduceStock` and `IncreaseStock` methods are instance methods that modify internal state (`StockQuantity`). In a concurrent environment (e.g., multiple users purchasing the last item simultaneously), external locking mechanisms or database-level concurrency tokens (such as ETags or RowVersions) are required to prevent race conditions and overselling. The class itself does not appear to implement internal synchronization.
*   **Null Collections**: The navigation properties `OrderItems` and `Reviews` are nullable. Consumers must check for null before iterating or counting these collections unless the ORM context is configured to initialize them automatically.
*   **Validation**: The stock modification methods imply strict guard clauses. Passing zero or negative values to `ReduceStock` or `IncreaseStock` will result in an exception; callers should validate user input prior to invoking these methods.
