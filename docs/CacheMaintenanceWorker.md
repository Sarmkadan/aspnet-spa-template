# CacheMaintenanceWorker

The `CacheMaintenanceWorker` is a background service responsible for periodic cache eviction, refresh, and health monitoring within an ASP.NET Core SPA application. It exposes metrics about the cache state (hit rate, item count, memory usage) and provides a `DefaultCacheHealthMonitor` property that can be used to obtain structured health reports. The worker runs continuously until cancelled, updating its internal status and warnings as it processes cache entries.

## API

### `public CacheMaintenanceWorker()`

Initializes a new instance of the `CacheMaintenanceWorker` class.  
No parameters are required. The worker is not started until `ExecuteAsync` is called.

### `public async Task ExecuteAsync()`

Starts the cache maintenance loop. This method should be called once, typically by the ASP.NET Core hosting infrastructure (e.g., via `IHostedService`).  
It runs indefinitely until the application’s cancellation token is triggered.  
**Throws:** `OperationCanceledException` when the cancellation token is signalled.

### `public BackgroundTaskStatus GetStatus()`

Returns the current status of the background maintenance task.  
**Returns:** A `BackgroundTaskStatus` value indicating whether the worker is running, stopped, or faulted.

### `public bool IsHealthy { get; }`

Indicates whether the cache is considered healthy based on the last health evaluation.  
**Returns:** `true` if the cache is healthy; otherwise `false`.

### `public double HitRate { get; }`

The ratio of cache hits to total lookups since the worker started.  
**Returns:** A value between `0.0` and `1.0` (inclusive). Returns `0.0` if no lookups have been performed.

### `public long ItemCount { get; }`

The current number of cache entries.  
**Returns:** The total count of items stored in the cache.

### `public long MemoryUsageBytes { get; }`

An estimate of the memory consumed by the cache, in bytes.  
**Returns:** The approximate memory footprint of cached data.

### `public int? WarningCount { get; }`

The number of active warnings generated by the cache maintenance logic.  
**Returns:** `null` if warnings have not been evaluated yet; otherwise the count of warnings.

### `public List<string> Warnings { get; }

A list of human-readable warning messages describing potential issues (e.g., high memory pressure, stale entries).  
**Returns:** A read-only list of warning strings. The list may be empty.

### `public DateTime GeneratedAt { get; }`

The timestamp of the last cache metrics snapshot.  
**Returns:** A `DateTime` in UTC indicating when the current metrics were recorded.

### `public DefaultCacheHealthMonitor DefaultCacheHealthMonitor { get; }`

Provides access to a `DefaultCacheHealthMonitor` instance that can produce detailed health reports.  
**Returns:** The health monitor object used by the worker.

### `public async Task<CacheHealthReport> GetHealthReportAsync()`

Generates a comprehensive health report for the cache.  
**Returns:** A `CacheHealthReport` containing status, metrics, warnings, and any detected issues.  
**Throws:** `InvalidOperationException` if the worker has not been started.

### `public async Task<bool> IsCacheHealthyAsync()`

Evaluates the current health of the cache asynchronously.  
**Returns:** `true` if the cache passes all health checks; otherwise `false`.  
**Throws:** `InvalidOperationException` if the worker has not been started.

## Usage

### Example 1: Starting the worker and checking basic metrics

```csharp
public class Startup
{
    public void ConfigureServices(IServiceCollection services)
    {
        services.AddSingleton<CacheMaintenanceWorker>();
        services.AddHostedService(sp => sp.GetRequiredService<CacheMaintenanceWorker>());
    }
}

public class CacheMonitorController : ControllerBase
{
    private readonly CacheMaintenanceWorker _worker;

    public CacheMonitorController(CacheMaintenanceWorker worker)
    {
        _worker = worker;
    }

    [HttpGet("cache/status")]
    public IActionResult GetCacheStatus()
    {
        var status = new
        {
            IsHealthy = _worker.IsHealthy,
            HitRate = _worker.HitRate,
            ItemCount = _worker.ItemCount,
            MemoryUsageMB = _worker.MemoryUsageBytes / (1024.0 * 1024.0),
            Warnings = _worker.Warnings
        };
        return Ok(status);
    }
}
```

### Example 2: Obtaining a health report and acting on it

```csharp
public class HealthCheckService
{
    private readonly CacheMaintenanceWorker _worker;

    public HealthCheckService(CacheMaintenanceWorker worker)
    {
        _worker = worker;
    }

    public async Task<bool> PerformCacheHealthCheckAsync()
    {
        if (_worker.GetStatus() != BackgroundTaskStatus.Running)
        {
            return false;
        }

        var report = await _worker.GetHealthReportAsync();
        if (!report.IsHealthy)
        {
            foreach (var warning in report.Warnings)
            {
                _logger.LogWarning("Cache health warning: {Warning}", warning);
            }
            return false;
        }

        return true;
    }
}
```

## Notes

- **Thread safety:** All public properties (`IsHealthy`, `HitRate`, `ItemCount`, `MemoryUsageBytes`, `WarningCount`, `Warnings`, `GeneratedAt`) are safe to read from multiple threads concurrently. They are updated atomically by the worker loop.  
- **Lifetime:** The worker must be started via `ExecuteAsync` before any health or metrics methods are called. Calling `GetHealthReportAsync` or `IsCacheHealthyAsync` before the worker is running will throw an `InvalidOperationException`.  
- **Warnings list:** The `Warnings` property returns a reference to the internal list. Consumers should treat it as read-only and not modify its contents.  
- **Memory usage:** `MemoryUsageBytes` is an estimate and may not reflect exact memory consumption due to garbage collection and object overhead.  
- **Cancellation:** `ExecuteAsync` respects the application’s shutdown cancellation token. The worker will stop gracefully when the token is signalled.  
- **Health monitor delegation:** The `GetHealthReportAsync` and `IsCacheHealthyAsync` methods delegate to the `DefaultCacheHealthMonitor` instance. The health monitor is created internally and should not be replaced.
