# Security Policy

## Reporting Security Vulnerabilities

We take security seriously and appreciate the community's help in identifying and reporting vulnerabilities responsibly.

### How to Report a Vulnerability

Please **do not** open public GitHub issues to report security vulnerabilities, as this could put the project and its users at risk.

Instead, use one of the following methods for reporting security vulnerabilities:

1. **GitHub Private Vulnerability Reporting**: Use GitHub's built-in private advisory feature.
2. **Email**: Contact rutova2@gmail.com with the details of the vulnerability.

### What to Expect

We are committed to addressing security vulnerabilities responsibly:

- We will acknowledge receipt of your vulnerability report within **48 hours**.
- We will assess the impact and provide a timeline for a fix and coordinated disclosure.

## Supported Versions

The following versions of the ASP.NET SPA Template are currently supported with security updates:

| Version | Supported |
| ------- | --------- |
| 1.x     | ✅ Yes    |
| < 1.0   | ❌ No     |
